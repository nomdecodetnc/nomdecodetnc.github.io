Stratégies d'usinage

Organiser les opérations par outil plutôt que par face

⏱ 8 min de lecture🏷 5 axes · Optimisation · Temps machine · Usure

En usinage 5 axes multi-faces, il existe deux façons d'organiser un programme : par face (on finit toutes les opérations d'une face avant de passer à la suivante) ou par outil (on garde l'outil en broche et on fait toutes les faces qui nécessitent cet outil avant de le changer). La deuxième approche est systématiquement plus efficace.

Organisation par face : le réflexe naturel

La majorité des programmes générés par FAO ou écrits sans réflexion sur l'optimisation sont organisés par face. C'est logique mentalement — on "pense" la pièce face par face. Mais en production, ça génère un nombre de changements d'outils disproportionné.

Exemple réel sur une pièce 5 axes avec 6 faces (Corps SW, plan 322-0504) :

  • Face B=0 (dessus) : T3 foret à pointer → T14 D2.8 → T15 taraud M3 → ...
  • Face B-90 : T3 foret à pointer → T14 D2.8 → T15 taraud M3 → ...
  • Face A+90 : T3 foret à pointer → T14 D2.8 → T15 taraud M3 → ...
  • ...

T3 (foret à pointer) est chargé et déchargé 9 fois pour pointer toutes les faces. 9 changements physiques pour le même outil.

Organisation par outil : l'outil reste en broche

La logique inversée : T3 est chargé une fois, tous les pointages de toutes les faces sont faits, puis T3 est déchargé définitivement. Le TNC 426 ne fait pas de changement physique si l'outil demandé est déjà en broche — même avec plusieurs TOOL CALL du même outil dans le programme.

* - OPERATION 1 : POINTAGE face B=0
TOOL CALL 3 Z S2000 F100 DL+0
; ... pointages face dessus ...

* - OPERATION 10 : POINTAGE face B-90
TOOL CALL 3 Z S2000 F100 DL+0   ; pas de changement physique — T3 déjà en broche
CALL LBL 103 ; INCLINAISON PLAN : B-90, DEC Y-16.5 Z-20
; ... pointages face B-90 ...

* - OPERATION 17 : POINTAGE face A+90
TOOL CALL 3 Z S2000 F100 DL+0   ; toujours pas de changement physique
; ... pointages face A+90 ...

Gains concrets et mesurables

Temps machine

Sur la pièce Corps SW : 59 changements physiques (version par face) → 31 changements (version par outil). 28 changements supprimés. À environ 15-20 secondes par changement sur DMU 70 : 7 à 9 minutes économisées par pièce. Sur une série de 25 pièces : 3 à 4 heures récupérées sans modifier une seule trajectoire.

Usure machine

  • Cône de broche : chaque changement d'outil sollicite le mécanisme de serrage
  • Bras changeur : cycles mécaniques supprimés
  • Glissières : moins de positionnements pour le changement
  • Électronique : moins de cycles d'accélération/décélération des variateurs

Réglage

Quand T3 est chargé une seule fois, le DL est mesuré une fois, rentré une fois, validé une fois. Dans la logique par face, T3 est rechargé après d'autres outils — une légère variation de serrage ou de mesure est possible à chaque rechargement. Moins de rechargements = moins d'occasions d'écart.

Risque d'erreur

Chaque rechargement d'outil est une occasion pour une erreur — mauvais outil, mauvais DL, mauvais DR. 28 rechargements en moins = 28 occasions en moins de faire une erreur.

Comment réorganiser un programme existant

La réorganisation ne touche pas aux trajectoires — uniquement à l'ordre des blocs d'opérations. La méthode :

  1. Identifier tous les TOOL CALL dans le programme
  2. Regrouper par outil : toutes les opérations T3, puis toutes les T14, etc.
  3. Pour chaque groupe, ordonner les faces dans un ordre logique (de la plus accessible à la moins accessible)
  4. Vérifier que les CALL LBL d'inclinaison de plan sont bien présents entre chaque changement de face
⚠️ Attention à la cohérence des opérations dépendantes : si un perçage doit être fait avant un taraudage sur la même face, ils doivent rester dans cet ordre même si on regroupe par outil. On regroupe d'abord tous les perçages (toutes faces), puis tous les taraudages (toutes faces) — jamais un taraudage avant le perçage correspondant.

Cette logique vient du décolletage

En décolletage sur tour multibroche ou tour automatique suisse, l'optimisation des temps de cycle est une contrainte absolue — chaque seconde de temps mort est de l'argent perdu. Les programmeurs décolletage pensent naturellement par outil, pas par face.

En fraisage 5 axes, cette rigueur est moins souvent appliquée — la complexité des trajectoires masque souvent l'inefficacité de l'organisation. Mais les gains sont réels et mesurables.

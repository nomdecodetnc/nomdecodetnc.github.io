Programmation Heidenhain TNC 426

L'en-tête de programme : la base d'un programme lisible

⏱ 5 min de lecture 🏷 TNC 426 · DMU 70 · Documentation

Un programme sans en-tête est un programme muet. Si un opérateur le retrouve dans 2 ans, il sera incapable de savoir quelle pièce il usine, quel est son système d'origines, si le brut est suffisant ou comment brider la pièce. L'en-tête résout tous ces problèmes en 30 secondes de lecture.

Ce que doit contenir un en-tête

Voici la structure minimale validée en production sur DMU 70 / TNC 426. Chaque champ a sa raison d'être.

; Date: 15/07/2026
;
; Machine: DMU-70
; Pro: 27032               <-- numéro interne de l'affaire
; Designation: CORPS SW
; Client: MAUNA KEA TECHNOLOGIES
; Plan: 322-0504 REV:A08   <-- révision du plan obligatoire
; Creation:
; Modification:
;
; OP programme/usinage: 1
; Dimention Brut:  114X84X48
; Dimention Fini:  112.107X83X40
;
; Origine Z: DESSUS PIECE
; Origine X: BUTER GAUCHE (quand mors fixe est au fond)
; Origine Y: MORS FIXE
;
; Hauteur de cales:
;
BLK FORM 0.1 Z X-0 Y-0 Z-40
BLK FORM 0.2 X+112.107 Y+83 Z+7
;
CYCL DEF 247 INIT. PT DE REF. ~
  Q339=16   ;NUMERO POINT DE REF.

Analyse champ par champ

Révision du plan

C'est le champ le plus critique. Un programme A07 utilisé sur une pièce A08 produit des pièces non conformes. Sans la révision, il est impossible de savoir si le programme est à jour ou obsolète.

⚠️ Ne jamais mettre en production un programme sans vérifier que sa révision correspond au plan actuel. En cas de doute, relire le plan et comparer les cotes critiques.

Dimensions brut ET fini

Les deux sont nécessaires. Le brut permet de vérifier avant de commencer si la matière disponible est suffisante. Le fini permet de comprendre ce que le programme est censé produire sans ouvrir le plan.

Origine expliquée en langage opérateur

C'est le point le plus souvent négligé. Écrire X0 Y0 CENTRE PIECE oblige l'opérateur à faire un calcul mental. Écrire BUTER GAUCHE (quand mors fixe est au fond) lui dit exactement où pointer sans ambiguïté.

✅ Règle : l'origine doit être compréhensible par un opérateur qui n'a jamais vu la pièce, sans avoir à ouvrir le plan.

Hauteur de cales

Si la pièce est bridée sur cales, leur hauteur détermine la position Z0. Ce champ, même vide, rappelle à l'opérateur de vérifier ce paramètre.

CYCL 247 — Initialisation du point de référence

Toujours mettre un CYCL DEF 247 en début de programme avec le numéro du point de référence utilisé. Cela garantit que le programme travaille dans le bon référentiel, même si quelqu'un a modifié la table entre temps.

CYCL DEF 247 INIT. PT DE REF. ~
  Q339=16   ;NUMERO POINT DE REF.

Ce que l'absence d'en-tête provoque

Situation Sans en-tête Avec en-tête complet
Reprise après arrêt long L'opérateur doit retrouver le plan, identifier la révision, recalculer les origines 30 secondes de lecture suffisent
Programme passé à un collègue Appel inévitable pour savoir comment brider Le collègue bride seul
Révision de plan Impossible de savoir si le programme est à jour sans comparer cote par cote La révision est visible immédiatement
Mauvais brut en stock Découverte au premier coup d'outil Détection avant de commencer

En-tête pour l'OP2 — particularités

En OP2 la pièce est retournée. L'origine X change de côté. Il faut l'indiquer explicitement pour éviter les erreurs de prise d'origine.

; Origine X: BUTER DROITE (quand mors fixe est au fond)
; Origine Y: MORS FIXE
; Origine Z: DESSUS PIECE

Si la même convention "BUTER GAUCHE" est utilisée en OP1 et en OP2 sans précision, un opérateur peut pointer du mauvais côté et casser la pièce.

💡 En 5 axes avec retournement, ajouter également CYCL DEF 8 IMAGE MIROIR si applicable, et l'indiquer dans l'en-tête pour que l'opérateur sache que le programme gère le miroir automatiquement.

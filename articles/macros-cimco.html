Macros CimcoEdit : organisation et passage de paramètres

⏱ 8 min de lecture🏷 CimcoEdit · Macros · Q-paramètres · PREPROG

CimcoEdit est un éditeur CN qui permet de créer des macros — des blocs de code réutilisables avec des champs variables. Bien organisées, ces macros permettent de générer un programme complet en quelques minutes. Mal organisées, elles créent plus de confusion qu'elles n'en résolvent.

Architecture d'une bibliothèque de macros

Une bibliothèque efficace est divisée en deux couches distinctes :

Couche 1 — Macros machine-spécifiques (MACRO000–MACRO119)

Ces macros sont organisées par groupes fonctionnels, spécifiques à la DMU 70 / TNC 426. Elles contiennent les instructions, cycles et conventions propres à cette configuration machine.

GroupePlageContenu
Infrastructure programme000–009En-tête, BLK FORM, CYCL 247, fin de programme
Dégagements 5 axes010–019LBL 90, LBL 91, LBL 92
Annulation plan020–029LBL 100
Inclinaisons standard030–049LBL 101–108 (B-90, B+90, A+90, A-90, etc.)
Cycles de perçage050–059CYCL 200, 207, 208, 201
Cycles de fraisage060–069CYCL 22, 25, 5
Surfaces et contours070–079LBL surfaçage, LBL contour générique

Divers DMU70100–119Opérations spécifiques, Q-paramètres initiaux

Couche 2 — Macros génériques (MACRO120–MACRO231)

Macros non liées à une machine spécifique — calculs, conventions ISO, blocs de commentaires, utilitaires.

Variables dans les macros CimcoEdit

Les variables dans une macro CimcoEdit sont locales à cette macro. Une variable $1 dans la macro A et une variable $1 dans la macro B sont deux variables indépendantes — elles ne se partagent pas.

Pour passer une valeur d'une macro à une autre, il faut utiliser le passage de paramètres par argument :

; Appel de la macro avec arguments :
; "insertmacro CYCL_PERCAGE 2 -15 100 0.6 0 0 50 0"
; Les arguments $1 à $8 sont passés dans l'ordre

; Contenu de la macro CYCL_PERCAGE :
CYCL DEF 200 PERCAGE ~
  Q200=$1    ;DISTANCE D'APPROCHE
  Q201=$2    ;PROFONDEUR
  Q206=$3    ;AVANCE PLONGEE
  Q202=$4    ;PROFONDEUR DE PASSE
  Q210=$5    ;TEMPO EN HAUT
  Q203=$6    ;COORD SURFACE
  Q204=$7    ;SAUT DE BRIDE
  Q211=$8    ;TEMPO AU FOND
💡 Cette approche permet de définir Q200, Q203, Q204 une seule fois dans la macro appelante et de les passer à toutes les macros de cycles — sans les retaper à chaque cycle.

PREPROG DMU70 — le template de programme complet

La macro PREPROG DMU70 génère le squelette complet d'un programme Heidenhain pour la DMU 70. Elle contient :

  • L'en-tête complet avec tous les champs à remplir
  • BLK FORM avec champs variables pour les dimensions
  • CYCL 247 avec numéro de point de référence
  • L'initialisation des axes (C+0 B+0 M5 M9)
  • Les labels d'infrastructure (LBL 90, 91, 92, 100, 101–108)
  • Le STOP M2 de fin de programme
  • La zone "labels de pièce" vide, prête à recevoir le contenu spécifique

En partant de PREPROG DMU70, on dispose d'un programme structurellement complet et conforme à la convention. Il ne reste qu'à ajouter les opérations spécifiques à la pièce.

La directive # dans CimcoEdit

Dans une macro CimcoEdit, le préfixe # est une directive d'écriture — il écrit le texte qui suit directement dans le fichier NC, sans le traiter comme une instruction macro. Les numéros de bloc sont ajoutés automatiquement par CimcoEdit et ne peuvent pas être modifiés manuellement (ce qui est correct — sur TNC 426, les numéros de bloc sont des étiquettes de navigation, pas des instructions fonctionnelles).
